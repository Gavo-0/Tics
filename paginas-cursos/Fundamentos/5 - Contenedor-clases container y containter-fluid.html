Bootstrap 5: Uso de container y container-fluid

En Bootstrap 5, todo diseño por rejilla (grid) parte de un contenedor que encierra filas (row) y columnas (col-*).

Bootstrap ofrece dos tipos principales de contenedores, según cómo deseas que se comporte respecto al ancho del navegador:

  • container (ancho fijo y centrado):
    • Tiene anchos fijos que cambian automáticamente según el breakpoint.
    • Siempre permanece centrado horizontalmente.
    • Ideal cuando el contenido no debe ocupar todo el ancho de pantalla, útil para mejorar la legibilidad o estética.
  • container-fluid (ancho 100% adaptable):
    • El contenedor se extiende al 100% del ancho del viewport.
    • Perfecto para diseños que necesitan aprovechar todo el espacio horizontal, como dashboards, apps o visualizaciones de datos.

Ejemplo práctico

El siguiente HTML muestra ambos tipos de contenedores y cómo se comportan: 

<!-- Contenedor con ancho fijo y centrado -->
<div class="container my-4">
  <h2>Contenedor Fijo (container)</h2>
  <div class="row">
    <div class="col-lg-4 bg-secondary text-white p-3">Columna 1</div>
    <div class="col-lg-4 bg-dark text-white p-3">Columna 2</div>
    <div class="col-lg-4 bg-secondary text-white p-3">Columna 3</div>
  </div>
</div>

<hr>

<!-- Contenedor fluido a todo lo ancho -->
<div class="container-fluid my-4">
  <h2>Contenedor Fluido (container-fluid)</h2>
  <div class="row">
    <div class="col-lg-4 bg-secondary text-white p-3">Columna 1</div>
    <div class="col-lg-4 bg-dark text-white p-3">Columna 2</div>
    <div class="col-lg-4 bg-secondary text-white p-3">Columna 3</div>
  </div>
</div>

¿Cómo funciona?

  • El sistema de bootstrap utiliza 12 columnas por fila.
  • Las clases col-lg-4 hacen que cada columna ocupe 4/12 unidades en pantallas grandes (≥992px).
  • En pantallas menores a 992px, las columnas se apilan verticalmente (responsive).
  • Ambos contenedores colapsan las columnas al mismo tiempo porque usan las mismas clases de columna.

¿Cuándo usar cada uno?

Caso de uso Usar container Usar container-fluid
Blogs, artículos, formularios Sí No recomendado
Dashboards, apps a pantalla completa No necesario Ideal
Estructura centrada con márgenes Sí No
SPA (Single Page App) Depende del diseño Generalmente sí

Observaciones clave y tips

  • Ambos tipos de contenedor pueden incluir cualquier componente de Bootstrap, no solo grids.
  • container tiene breakpoints internos y su ancho se adapta automáticamente según el tamaño de pantalla, pero siempre está centrado.
  • container-fluid ocupa siempre el 100% del ancho de la ventana, sin márgenes laterales.
  • Puedes anidar containers, pero normalmente solo se usa un container principal por página.
  • Usa container para contenido legible y container-fluid cuando necesitas aprovechar toda la pantalla.

¿Cómo probarlo?

  • Abre la página en tu navegador y cambia el tamaño de la ventana para ver cómo el container mantiene un ancho fijo y centrado, mientras que container-fluid se adapta siempre al 100%.
  • En dispositivos móviles, ambos containers colapsan las columnas de igual forma, porque usan col-lg-4.
